This document contains a comprehensive analysis of the `home_workout_app` project (branded as **Lumina Fitness**). The report is structured to fulfill college internship documentation requirements, including architectural deep-dives, screen catalogs, and log activities.

## 2. One Line Summary
An offline-first, glassmorphic Flutter fitness companion that delivers interactive workout routines with real-time bilingual voice coaching, streak challenges, and progress trackers.

---

## 3. Detailed Description
Lumina Fitness is designed as a personal home training companion that eliminates the need for expensive gym memberships or complex exercise hardware. By targeting bodyweight routines, the application guides users through exercises with step-by-step stance instructions, animated visual indicators, and vocal prompts. To motivate users, the system incorporates a 30-day habit streak challenge and a local multiplayer leaderboard. The interface adopts a modern "Liquid Glass" glassmorphism aesthetic that flows over ambient glowing backdrops, ensuring a premium user experience.

---

## 4. Features

### Completed Features
* **Bilingual Voice Coaching:** Real-time Text-to-Speech (TTS) coaching cues in English and Hindi using [TtsService](file:///C:/FLUTTER/home_workout_app/lib/services/tts_service.dart#L5).
* **Onboarding Configuration:** A wizard capturing biological details (gender, age, height, weight) with toggleable metric/imperial units.
* **30-Day Streak Challenges:** Dynamic checking engines supporting 7-day, 14-day, and 30-day habit sprints with automatic day tracking.
* **Interactive Progress Analytics:** Dynamic weight logging and active calorie tracking visualized on charts via `fl_chart`.
* **Workout Timer Engine:** A state-driven active session tracker managing prep phases, exercise intervals, rest countdowns, and sound indicators.
* **Custom Workout Plan Creator:** An interface allowing users to curate custom routines, specify durations, and arrange exercises.
* **Liquid Glass Visual Engine:** Glassmorphism overlay rendering 3D meniscus borders, crescent sheens, and glare highlights on top of content cards.
* **Hydration Tracker:** A daily logging dashboard for logging water consumption.
* **Form Coach Collapser:** Stance instruction panels and execution warnings embedded inside the workout detail stream.

### Partially Implemented Features
* **Mock User Authentication:** Sign-up and login logic are mock-simulated inside [AuthService](file:///C:/FLUTTER/home_workout_app/lib/services/auth_service.dart#L4) returning static objects, rather than connecting to a live secure API gateway.
* **Local-only Leaderboards:** Multiplayer ranks are generated locally with static competitors (e.g. Ryan Reynolds, Emma Watson) rather than sync-fetching from a multiplayer server database.

## 5. User Flow
1. **Startup & Splash:** The app launches on [splash_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/splash_screen.dart), checking Hive local storage for a cached user session. If absent, the user is routed to onboarding.
2. **Onboarding & Profiling:** The user completes the profiling sequence, configuring gender, age, height, weight, fitness goals, and training preferences.
3. **Home Dashboard:** The user lands on [home_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/home_screen.dart) displaying a personalized time-based greeting, active streak counts, a "Start Today" workout panel, quick shortcuts, and the 30-day progress grid.
4. **Selecting a Plan:** Tapping a workout opens the details sheet. The user reviews difficulty ratings, required equipment, and exercise descriptions.
5. **Active Workout Execution:** Tapping "Start" launches [active_workout_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/active_workout_screen.dart). The countdown timer guides the user, playing audio cues and rendering stance animations. The user can pause, resume, or skip movements.
6. **Summary Celebration:** Upon completion, the user is redirected to [summary_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/summary_screen.dart), triggering confetti, tracking burned calories, and updating history.
7. **Streaks & Leaderboard Check:** The user opens the Challenges tab to view check-ins and check their rank on the leaderboard.
8. **Logging Stats:** The user logs their weight and water intake, viewing progress charts on the Progress tab.

## 7. Architecture

### Folder Structure
The codebase resides under the standard [lib](file:///C:/FLUTTER/home_workout_app/lib/) directory:
* `config/`: Hosts global setups (e.g. [app_config.dart](file:///C:/FLUTTER/home_workout_app/lib/config/app_config.dart)).
* `data/`: Local raw datasets loaded into memory.
* `models/`: Plain Dart entities defining properties (e.g. [exercise.dart](file:///C:/FLUTTER/home_workout_app/lib/models/exercise.dart)).
* `providers/`: State management controllers managing updates (e.g. [progress_provider.dart](file:///C:/FLUTTER/home_workout_app/lib/providers/progress_provider.dart)).
* `screens/`: Hosts the 48 UI components and workflow containers.
* `services/`: Encapsulates operational logic (e.g. [tts_service.dart](file:///C:/FLUTTER/home_workout_app/lib/services/tts_service.dart)).
* `theme/`: Dictates the visual styling rules.
* `widgets/`: Houses reusable custom design controls (e.g. [custom_card.dart](file:///C:/FLUTTER/home_workout_app/lib/widgets/custom_card.dart)).

### State Management
* **Framework:** **Flutter Riverpod**
* **Implementation:** Employs Riverpod's `Notifier` and `NotifierProvider` classes for dynamic state updates, combined with `FutureProvider` for asynchronous asset loading (e.g. `workoutPlansProvider`).

### Navigation System
* **Framework:** Imperative routing using Flutter's native `Navigator` API (`Navigator.push` / `Navigator.pop`).
* **Main Container:** Host tabs are managed in an `IndexedStack` inside [main_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/main_screen.dart) to preserve screen states during tab swaps.

### Database / Storage
* **Database Engine:** **Hive** and **Hive Flutter**
* **Implementation:** Two primary storage boxes:
  * `userProgressBox`: Stores the [UserProgress](file:///C:/FLUTTER/home_workout_app/lib/models/user_progress.dart#L1) object (streaks, completed day lists).
  * `settingsBox`: Stores calorie/weight logs, checked-in challenge dates, custom plans, theme indexes, and timer presets.

### APIs & External Services
* **REST Networking:** Uses the `http` package to validate API credentials against an authentication gateway.
* **Audio Voice Services:** Interfaces with the device native Text-To-Speech systems via `flutter_tts`.
* **Haptics:** Direct triggers to system tactile engines via `HapticFeedback` inside [haptic_service.dart](file:///C:/FLUTTER/home_workout_app/lib/services/haptic_service.dart#L3).

## 9. Data Flow
1. **Asset Fetching:** On startup, [dataProvider](file:///C:/FLUTTER/home_workout_app/lib/providers/data_provider.dart) fetches `exercises.json` using `rootBundle.loadString`, parsing it into list objects (`List<WorkoutPlan>` and `List<Exercise>`).
2. **State Watchers:** Widgets watch [progressProvider](file:///C:/FLUTTER/home_workout_app/lib/providers/progress_provider.dart#L386) to rebuild components in response to streak changes, weight logs, or challenge updates.
3. **Session Start:** Starting a routine calls `startWorkoutSession(...)` inside [ProgressNotifier](file:///C:/FLUTTER/home_workout_app/lib/providers/progress_provider.dart#L58), creating a placeholder session labeled `status: partial` in Hive.
4. **Active Workout State Loop:** [active_workout_screen.dart](file:///C:/FLUTTER/home_workout_app/lib/screens/active_workout_screen.dart) listens to [workoutProvider](file:///C:/FLUTTER/home_workout_app/lib/providers/workout_provider.dart#L191). The provider runs a periodic timer, decrementing duration every second, updating the state, and prompting `TtsService` to speak cue strings.
5. **Workout Complete:** On completion, the system:
   * Calls `completeWorkoutSession(...)` to update the history session status to `completed` in Hive.
   * Logs burned calories (~15 cal per exercise).
   * Marks the day completed, increments streak counts, and pushes updates to the leaderboard.
6. **Persistence:** All progress notifier operations execute sync updates directly to the Hive database boxes to prevent data loss.

## 12. Internship Report Content

### Introduction
Mobile applications have emerged as key tools for personal wellness. Lumina Fitness is a mobile fitness client developed using Flutter. It is designed to act as an offline-first fitness coach, providing structured training regimens without requiring specialized equipment.

### Objective
The primary goal of this project is to build a highly responsive home workout tracking platform that encourages physical consistency. Key objectives include:
* Developing a lightweight, local-storage database engine using Hive.
* Building a real-time voice coaching system supporting dual-language outputs (English and Hindi).
* Constructing an interface using glassmorphic design principles.
* Implementing progress tracking through weight and calorie charts.

### Problem Statement
Most modern fitness applications require expensive premium tiers, constant internet connectivity, and complex external tracking hardware. This creates a barrier for users seeking a simple training guide. Furthermore, many applications feature flat, generic designs that fail to engage the user. Lumina Fitness addresses these issues by offering an offline-first training guide with high-fidelity, interactive glassmorphic visuals and bilingual audio cues.

### Methodology
The development process followed an agile, feature-driven approach:
* **Research and Setup:** Analyzed requirements and structured the project directories.
* **Onboarding Setup:** Built a configuration wizard to gather profile details.
* **Database Integration:** Configured Hive storage to cache progress and settings.
* **Core Workouts and TTS:** Implemented the active workout timer and Text-to-Speech coaching engine.
* **Design Implementation:** Developed the Liquid Glass visual theme using a custom painter overlay.
* **Testing and Refactoring:** Ran code analysis (`flutter analyze`) to resolve issues.

### Features
* **Bilingual Voice Alerts:** Speaks instructions in English and Hindi.
* **Streak Checking:** tracks consistency using 7/14/30 day rings.
* **Tactile Feedback:** Integrates haptic feedback on button presses.
* **Analytical Progress Charts:** Visualizes weight changes and calorie logs.
* **Animated Exercise Form:** Plays Lottie JSON vectors to guide exercises.

### Future Scope
* **Cloud Sync Integration:** Connecting backend server databases (e.g. Firebase, PostgreSQL) to sync user profiles.
* **Social Multiplayer Features:** Enabling real multiplayer leaderboards and progress sharing with friends.
* **Wearable Device Integration:** Connecting with smartwatches and health platforms (e.g. Google Health Connect, Apple HealthKit).

### Conclusion
Lumina Fitness provides a robust home workout tracking experience. By combining local NoSQL caching, a state-driven workout timer, bilingual voice guidance, and a modern glassmorphic interface, the project demonstrates how Flutter can be used to build premium, responsive, and accessible health applications.
